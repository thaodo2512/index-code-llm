The big idea — N repos in, one endpoint out

🧭

Multiple repos → one endpoint

Register N repos; query them through a single MCP server with cross-repo search and per-repo tools.

🛰️

Remote by design

Index heavy trees on a beefy server; query from a laptop over SSH. No open ports, no daemons.

🧼

Zero repo footprint

Your repos are never modified — no .codegraph/, no .gitignore edits, clean git status.

Architecture — one gateway, one CodeGraph child

The trick — non-git symlink views

Why: CodeGraph lists tracked files with git ls-files (un-scopeable) — but a non-git dir triggers a filesystem walk that follows symlinked dirs. The view exposes only your chosen subtrees, so the index is exactly scoped and the repo stays byte-for-byte pristine.

Setup — describe · build · index · serve

1 · Declare the workspace — workspace.json

{
  "viewsDir": "/data/views",
  "repos": [
    { "name": "linux",  "path": "/repos/linux",
      "scope": ["kernel", "include"] },
    { "name": "zephyr", "path": "/repos/zephyr",
      "scope": ["kernel", "include"] },
    { "name": "myapp",  "path": "/repos/myapp",
      "scope": ["."] }
  ]
}

scope = subtrees to index · ["."] = whole repo · keep viewsDir outside every repo

2 · What gets materialized on disk

📁 /data/views/                # NOT inside any repo
├─ 📁 linux/
│  ├─ 🔗 .gitignore → repo's
│  ├─ 🔗 kernel     → /repos/linux/kernel
│  ├─ 🔗 include    → /repos/linux/include
│  └─ 🗄️ .codegraph/codegraph.db
└─ 📁 zephyr/ …

3 · Four commands

$ codegraph-workspace build-views
$ codegraph-workspace index
$ codegraph-workspace sync     # cron / timer
$ codegraph-workspace serve --stdio | --http

Deploy — full Docker, guided scripts

./scripts/local_deploy.sh  💻

Everything on your laptop — the client connects over localhost. Optional bearer token. Port busy? It offers the next free one.

./scripts/remote_deploy.sh  🛰️

Indexes on a powerful server, queried over an SSH tunnel. Checks key auth (offers ssh-copy-id), copies the project, builds, indexes, wires the tunnel + token.

The tool surface

ToolWhat it does
workspace_search Fuzzy symbol search across all repos (or a subset). Ranked hits, each tagged with its repo — the cross-repo find.
workspace_repos Repos + index stats (files / nodes / edges / size). Cheap by default; includeFreshness reports un-synced changes.
ws_contextws_tracews_explore
ws_callersws_calleesws_impact
ws_nodews_files 		CodeGraph's rich per-repo tools, each taking a repo argument and routed to the live child process.
read-only by design No write or exec tools — an agent can explore code through this server, never modify it.

One compact tool set with a repo arg = low token cost, full reach. Works with Claude Code, Codex, Cursor, opencode, Hermes — any MCP client.
